# RedBox 桌面端更新日志

## v2.1.0（2026-05-20）

### 1）新增：数字人视频生成链路
- 新增数字人视频生成能力，支持用角色参考视频和配音音频生成口播视频，适合固定 IP、讲解视频和账号内容复用。
- 角色库支持保存参考视频，数字人创作可直接读取角色的视频、声音和基础设定，不需要每次重新选择素材。
- 数字人生成已接入媒体任务系统，支持提交任务、等待结果、解析成片地址和回到创作工作台查看结果。
- 数字人工作流并入生成工作台，入口更集中，视频、音频、角色资产和生成结果可以在同一条创作链路中处理。
- 支持基于角色视频进行音色克隆，数字人流程可以先生成更匹配角色的配音，再进入视频合成。

### 2）增强：TTS 与音色选择
- 音频生成新增按语言筛选音色，选择语言后只展示更适合当前语言的系统音色和已克隆音色。
- 音色选择改为跟随 TTS 模型变化，不同模型只展示对应可用的音色，减少选错模型或音色后生成失败的情况。
- 修复声音克隆槽位和模型绑定关系，角色可以更稳定地保存多个模型下的音色配置。
- 修复 TTS 模型列表来源，避免页面展示的模型与实际可调用能力不一致。

### 3）新增：知乎内容采集与知识库接入
- 浏览器插件新增知乎内容采集能力，可将知乎页面内容送入 RedBox 知识库，用于后续检索、分析和创作。
- 修复知乎采集路由与知识库索引路径，采集后的内容可以更稳定地进入知识库并参与检索。
- 知识库空状态新增插件安装引导，首次使用外部采集时更容易找到正确入口。

### 4）增强：聊天与媒体素材处理
- 聊天输入框支持直接粘贴图片，粘贴后的图片会作为附件进入当前对话和后续 Agent 工具调用。
- 媒体结果新增保存相关操作，生成后的图片、音频、视频更容易沉淀为后续可复用素材。
- 支持更多 Windows 本地媒体资产路径，减少本地图片、音频、视频在预览和生成链路中无法访问的问题。

### 5）新增：拖拽文件快捷功能
- 支持将本地文件直接拖入聊天窗口，快速作为附件加入当前对话。
- 拖入图片、视频等媒体文件后，会自动进入后续预览、分析和 Agent 工具调用链路。
- 浏览器插件支持拖拽网页图片保存到 RedBox 素材库，减少从网页素材到本地素材库的手动下载步骤。

![拖拽文件快捷功能演示](https://github.com/Jamailar/RedBox/releases/download/v2.1.0/redbox-2.1.0-drag-file-shortcut-demo.gif)

### 6）修复：稳定性与工作流
- 修复知识库视频转写重试逻辑，减少上游临时失败导致视频资料无法入库的问题。
- 修复聊天快捷提示词被显示到正式消息里的问题，保持对话内容更干净。
- 修复已有 RedClaw 空间无法进入新样式流程的问题，旧空间可以继续使用新的创作链路。
- 修复官方 AI 设置中的缓存用量显示问题，减少账号用量状态和页面展示不一致的情况。

## v2.0.0（2026-05-14）

### 1）新增：声音复刻与角色克隆
- 新增声音复刻能力，可基于参考音频生成更贴近原声线的配音资产，适合固定栏目、口播账号和系列化内容生产。
- 新增角色克隆链路，支持沉淀角色形象、声音与风格设定，让同一 IP 在多轮创作中保持更稳定的一致性。

### 2）增强：视频生成接入 Seedance 2.0
- 视频生成模型接入 Seedance 2.0，提升画面质感、运动连贯性和短视频成片稳定性。
- 生成工作台的视频能力升级为更适合自媒体生产的主路径，可围绕脚本、角色、声音和视频结果连续迭代。
- 新增视频分析与剪辑能力，支持拆解爆款视频结构、识别高光片段，并剪辑成可复用的精彩切片。

### 3）升级：全新 UI 体验
- 重新整理核心创作界面，减少视觉噪音，让素材、生成、任务和结果管理更直观。
- 优化常用操作的入口层级，降低从想法到成片的路径成本。

### 4）增强：更稳定的 Agent 系统
- Agent 系统稳定性继续增强，任务规划、工具调用、上下文传递和结果汇总更可靠。
- 整体能力对标自媒体版 Codex，更适合处理选题、脚本、素材、视频生成和发布准备这类连续创作任务。

## v1.10.4（2026-04-27）

### 1）新增：插件侧边栏社媒采集与知识库接入
- 浏览器插件新增侧边栏采集工作台，支持从小红书、抖音、B 站、快手、TikTok、Instagram、Reddit、X 等平台识别内容并送入 RedBox。
- 插件补齐平台图标、设置页、页面观察器和小红书桥接层，长内容采集不再只依赖单一 popup 入口。
- 知识库新增 social capture 类型的条目、索引字段和加载路径，外部平台采集内容可以进入桌面端资料库并参与后续检索。
- 知识库页面同步支持展示和筛选社媒采集来源，便于把外部素材直接转成创作资料。

### 2）新增：Wander 手动素材选择与稿件标题编辑
- Wander 新增引导式手动素材选择，用户可以在自动推荐之外主动挑选资料，再进入后续创作流程。
- 桌面端新增对应的 Wander host command，素材选择结果通过结构化数据返回，避免只靠页面临时状态拼接。
- 稿件列表支持行内编辑标题，标题修改不需要离开当前列表或打开额外编辑弹窗。

### 3）增强：封面参考图与生成附件链路
- 封面编辑图片请求改用兼容字段名，并切换到内置 multipart 图片请求实现，减少不同模型接口之间的字段不一致。
- 参考图下载不再依赖本机 `curl`，可直接由宿主侧下载并处理远程图片，降低用户环境差异带来的失败率。
- 修复 `file://` 参考图解析与工作区资产路径迁移后的重定位，封面生成可以更稳定地使用本地图片和迁移后的素材。
- 参考图角色说明更清晰，区分主体、风格、构图等不同用途，降低封面生成时的输入歧义。
- 修复生成工作台 Agent 内联附件传递，聊天里的图片附件可以更可靠地进入宿主侧生成流程。

### 4）增强：CLI / MCP 有效环境与工具曝光
- CLI Runtime 修复宿主路径探测，补齐 shell 环境、PATH、解释器闭包与 sandbox 变量读取，工具执行更接近真实本机终端环境。
- MCP 调用同步使用有效环境，减少外部 server 因 PATH、HOME、解释器路径不一致而启动失败的问题。
- 工具目录与兼容层修复 host tool 曝光问题，Agent 可以更稳定地看到并调用应当可用的宿主能力。
- 新增 CLI / MCP 有效环境优化计划文档，记录环境收集、执行注入、安全边界和验证策略。

### 5）修复：编辑器选择、RedClaw 展示与数据迁移
- 修复稿件编辑器文本选中、复制与样式作用域问题，避免编辑区被外层选择规则干扰。
- RedClaw 工作流展示优化，聊天和任务页里的执行进度更容易看清当前步骤。
- 修复工作区迁移后的本地资产路径重定位，旧资料、稿件和素材搬迁后可以继续被正确加载。
- 扩展旧版导入和 workspace loader 兼容逻辑，降低历史数据迁移后的资源丢失概率。

## v1.10.3（2026-04-26）

### 1）新增：结构化工具路由与运行计划
- 新增会话级工具目录与工具计划，运行时会按当前任务动态暴露可用 action，减少一次性暴露过多工具带来的选择噪音。
- 工具调用增加 turn context、plan checkpoint、plan fingerprint 与执行快照，长任务中每次调用都能带上更明确的上下文和回放线索。
- 新增 action search / deferred action discovery，Agent 可以先搜索能力再调用具体 action，不再依赖硬编码工具清单。
- 工具家族新增 exposure policy 与 metadata policy，可按任务限制文件、编辑器、团队、记忆、媒体生成等能力的可见范围。
- 对延迟工具、App action 和兼容别名增加校验与拒绝路径，避免不该被当前 runtime 调用的能力漏进执行链路。

### 2）新增：MCP 工具目录、资源与团队适配层
- 新增 MCP server 配置、manager、resources、tool inventory、tool exposure 与工具命名规范，桌面端可以更稳定地管理外部工具服务。
- MCP 工具支持按 server / tool 生成清晰目录，并把工具描述、输入 schema、可见策略和资源信息纳入统一运行时。
- Team MCP adapter 继续增强，团队协作能力可以通过结构化工具入口被发现和调用，而不是只依赖聊天文本约定。
- MCP 工具配置补齐 required、approval、timeout、active-time 等运行约束，为后续更多外部工具接入打好边界。

### 3）新增：成员技能、顾问蒸馏与知识范围绑定
- 新增成员技能蒸馏技能，可以把顾问能力、知识范围和协作偏好沉淀成可复用的成员技能。
- 成员技能支持候选生成、版本管理、推广校验、回滚、重试和开关配置，避免一次生成后无法修正。
- 成员可以绑定知识范围与语言范围，技能匹配时会优先考虑专长、可用工具、知识权限和当前负载。
- 新增成员 profile / agent card 与成员匹配流程，让团队分工可以按能力卡片选择成员，而不是只看显示名称。
- 顾问相关页面和宿主命令补齐了能力沉淀入口，方便把已有专家配置迁移为长期可维护的成员能力。

### 4）增强：协作工作台与团队任务闭环
- 协作工作台补齐项目控制、任务控制和工作板操作，任务可以在桌面端更直接地创建、推进和查看状态。
- 新增任务评论、执行报告保留、成员 mailbox 清理和团队唤醒修复，减少多人协作时的状态丢失和旧消息残留。
- 协作 runtime 增加 verifier 闭环、交付证据、完成声明和 handoff packet，任务完成会先经过结构化校验再进入总结。
- 团队成员的发言者与执行者共享同一 profile，执行进度通过 plan、event、artifact 和 speech queue 回到群聊，避免“说”和“做”脱节。
- 协作运行时文档同步标记完成状态，并补齐团队消息、任务事件和工作台契约，后续排障更容易对照实现。

### 5）增强：记忆、知识库与检索稳定性
- 新增 indexed memory recall 能力，记忆可以进入本地索引并按关键词、语义线索和上下文进行召回。
- 知识库支持按成员知识范围建立索引，并在检索排序中考虑语言匹配，减少中文资料和英文查询混用时的偏差。
- canonical cache 增加复用与 parser / OCR 版本校验，文档重解析时可减少重复工作，同时避免旧解析结果被误用。
- OCR 重建改为显式触发，知识库刷新不会因为隐式重建拖慢页面或制造不可预期的后台工作。
- 修复知识目录刷新延迟、索引迁移状态、分块文档记录和工作区加载兼容问题，提升大知识库下的刷新稳定性。

### 6）优化：过程展示、提示词叠加与设置页
- 聊天过程恢复内联思考摘要，并把工具状态压缩为更轻量的逐轮进度文本，减少长任务执行时的跳动和噪音。
- 工具状态支持显示技能名称、折叠旧状态、合并工具进度，回放任务时更容易看清“当前在做什么”。
- runtime overlay 重新整理：子代理角色、媒体规则、RedClaw 规则和核心运行规则进入更清晰的系统叠加层，减少重复 prompt。
- 设置页精简高级设置和通知设置，移除手动知识索引按钮，并屏蔽部分计划中 API 的告警噪音。
- 生成工作台会读取已配置的图片模型选项，图片 runtime 也补齐媒体生成工具曝光，模型配置与页面入口更一致。

### 7）修复：更新、插件、导入与模型兼容
- 恢复自动更新检查，避免桌面端启动后漏掉可用版本。
- 修复插件桥接大体积小红书采集时的数据传输问题，降低长内容采集失败概率。
- 修复 RedClaw 路由消息未创建会话的问题，避免从外部入口进入时丢失执行上下文。
- 扩展旧版 store 导入覆盖范围，迁移历史数据时可以识别更多旧结构。
- 修复文件型历史记录计数、部分模型接口地址规范化、显式网页读取和运行时恢复路径中的兼容性问题。

- 工具层新增执行输出快照与更完整的 catalog 暴露，让 Agent 在调用 CLI 时能保留更稳定的执行上下文和回放信息。

- 稳定高并发媒体任务调度与后台任务处理，新增异步轮询器并修复大批量任务下的排队、恢复与状态同步问题。
- 修复会话历史复用时的陈旧 tool protocol 消息污染、孤儿 tool result 回放、聊天室成员 runtime 残留等问题，减少 Agent 状态错乱。
- 修复记忆列表死锁、Subjects 创建刷新与属性编辑、Windows 媒体预览路径归一化、桌面语音转写等多个兼容性问题。

  - 生视频走视频模型链路并支持多模型回退。
- 建立任务输入标准化结构，包含：文案稿件引用、主体信息、尺寸配置、风格配置、附加提示词、预算/优先级。
- 新增生成任务生命周期事件（排队、执行中、失败重试、完成），让页面可直接渲染实时进度并可中断/重试。
- 支持在媒体库一键关联稿件上下文：
  - 成果文件落库到稿件目录。
  - 缩略图和元信息自动回写便于后续封面/导出复用。

### 2）增强：Agent CLI 调用能力
- 新增/增强 Agent 对「CLI 工具调用」的结构化调用能力，支持在任务中安全触发本地命令执行。
- 明确了 CLI 工具边界与参数约束：
  - 独立 tool/action 定义，不再用自然语言文本做硬匹配。
  - 使用 typed payload（参数结构体）定义执行域、超时、重试策略、输出抓取模式。
- 完善权限与安全防线：
  - 执行命令执行前做白名单校验与参数白名单过滤。
  - 增加错误码与标准化异常返回（非零返回、超时、权限拒绝三类可区分）。
- 运行时改造：
  - CLI 执行从事件总线触发，保持 Agent 逻辑与执行器解耦。
  - 命令执行日志写入可追踪字段（任务 ID、runtime mode、invocation trace）。

### 3）增强：知识库检索能力
- 重构知识检索入口为显式检索管道：
  - 关键词预检 → 向量召回 → 结果重排序 → 上下文打包。
- 引入检索评分可观测化：
  - 记录召回 score、rerank score、命中文档命中原因。
  - 支持按时间范围、标签、工作区、内容类型做二次过滤。
- 增强索引回填效率：
  - 生成时先做 embedding 缓存，查询时复用热缓存。
  - 支持分页增量更新与分片重建，避免全库阻塞。
- 支持 Agent 在创作页自动拉取知识引用块，降低 hallucination，并把引用源回填到任务元数据。

### 发版建议（执行层）
- AI 侧：将模型与生成链路统一放到 runtime skill/prompt 边界，避免在页面里写关键词路由。
- 生成侧：优先复用统一 `runtime:*` IPC + `bridge/ipcRenderer.ts`，避免分散调用。
- 性能建议：
  - 生图/生视频任务采用异步队列 + 并发阈值（默认 2/CPU）；
  - 知识检索开启短路缓存（最近 30s 热查询）与 TTL 保护；
  - 大列表分页加载，优先展示 skeleton + 增量刷新，保持首屏不被 IPC 阻塞。
