# RedBox 桌面端更新日志

## v1.10.2（2026-04-25）

### 1）新增：法律级检索与索引重建能力
- 知识检索新增 BM25 词法索引、Tantivy 块级检索索引与可插拔 reranker，检索链路从单一路径扩展为可组合、可观测的召回与重排流程。
- 文档解析管道支持可插拔 parser / OCR provider，并补齐索引迁移、分层重建、重建审计与升级生命周期，降低大规模知识库升级时的中断风险。
- 设置页新增检索服务配置入口，便于在桌面端直接查看和调整检索能力，而不是依赖隐藏配置或手工改文件。

### 2）增强：本地 CLI Runtime 与工具可观测性
- 本地 CLI 发现与兼容层继续增强，补齐命令别名诊断、解释器闭包解析、macOS sandbox 系统配置读取兼容，以及执行模式持久化。
- 工具层新增执行输出快照与更完整的 catalog 暴露，让 Agent 在调用 CLI 时能保留更稳定的执行上下文和回放信息。
- 设置页同步收敛 CLI 运行时安全项与执行模式展示，降低配置改动后丢失状态或行为不一致的问题。

### 3）增强：协作工作台与 Team Runtime
- 协作工作台补齐 ACP 控制入口、项目创建控制与本地 CLI 动态发现，团队协作链路更容易在桌面端直接启停和调试。
- Team Runtime 对 AI 暴露了更多 coordinator action，并增加 RedBox team MCP adapter contract，方便把团队能力接入到结构化工具目录中。
- 同时移除了外部 ACP agent path 方案，减少历史兼容分支和多入口并存造成的运行时复杂度。

### 4）修复：设置、资源发现与 RedClaw 稳定性
- 修复 AI 设置持久化竞争、打包后 builtin resource 发现异常，以及 RedClaw onboarding 回答提交时的竞态问题。
- 修复设置页和 Wander 页之间的部分联动异常，减少运行模式、官方支持配置与页面状态不同步的问题。

## v1.10.1（2026-04-25）

### 1）新增：RedClaw 任务化创作链路
- 完成 RedClaw 任务管理统一，创作动作从零散入口收敛为明确的任务操作，便于批量执行、继续处理和失败恢复。
- 新增 RedClaw 引导流宿主与共享 onboarding 结构，把首次配置、任务进入和风格初始化放到同一条主路径里，减少跳转和状态丢失。
- 强化图片生成默认策略与批处理卡片自动执行逻辑，降低首轮配置成本，让图像任务可以更稳定地直接跑起来。

### 2）增强：生成工作台 Agent / Suite 工作流
- 套图 Agent 正式留在生成工作台内运行，不再切出独立窗口；会话元数据、执行转录和历史记录会跟随生成上下文一起持久化。
- 补齐多模态输入链路，支持在工作台中传递内联图片上传与 `data URL` 附件，减少图片创作过程里的上下文断裂。
- 增加 Agent 规划护栏、旧版历史兼容恢复、批次路由快照与六图上限控制，降低长任务和连续创作时的异常概率。

### 3）增强：CLI 诊断、设置与创作者配置
- 新增 CLI discover / resolve 诊断能力，并在设置页暴露可观察入口，便于快速定位本地命令缺失、别名兼容和 PATH 异常。
- 设置页支持按工作区编辑创作者资料，并修复资料切换后的刷新与回填问题，让内容配置与当前工作区保持一致。
- 写作风格与任务技能默认收敛到 turn 级作用域，减少隐式激活带来的串场问题，保证单轮任务边界更清晰。

### 4）新增：通知中心与音频输入能力
- 新增通知中心基础能力，统一管理运行完成提示、系统通知、提示音与动作路由，减少后台任务完成后的信息遗漏。
- 增加音频命令与录音输入链路，优化语音转写处理，让语音输入结果更稳定地进入对话和创作流程。

### 5）修复：稳定性、调度与兼容性
- 稳定高并发媒体任务调度与后台任务处理，新增异步轮询器并修复大批量任务下的排队、恢复与状态同步问题。
- 修复会话历史复用时的陈旧 tool protocol 消息污染、孤儿 tool result 回放、聊天室成员 runtime 残留等问题，减少 Agent 状态错乱。
- 修复记忆列表死锁、Subjects 创建刷新与属性编辑、Windows 媒体预览路径归一化、桌面语音转写等多个兼容性问题。

## v1.9.4（2026-04-24）

### 1）新增：创作页面（图文 / 生图 / 生视频）
- 新增「创作工作台」主入口，支持统一进入图文、图生图、图生视频生成链路，打通稿件与媒体资产的上下文。
- 实现模型路由策略：
  - 生图默认优先接入 `gpt-image-2`，保留现有 fallback 模型兜底。
  - 生视频走视频模型链路并支持多模型回退。
- 建立任务输入标准化结构，包含：文案稿件引用、主体信息、尺寸配置、风格配置、附加提示词、预算/优先级。
- 新增生成任务生命周期事件（排队、执行中、失败重试、完成），让页面可直接渲染实时进度并可中断/重试。
- 支持在媒体库一键关联稿件上下文：
  - 成果文件落库到稿件目录。
  - 缩略图和元信息自动回写便于后续封面/导出复用。

### 2）增强：Agent CLI 调用能力
- 新增/增强 Agent 对「CLI 工具调用」的结构化调用能力，支持在任务中安全触发本地命令执行。
- 明确了 CLI 工具边界与参数约束：
  - 独立 tool/action 定义，不再用自然语言文本做硬匹配。
  - 使用 typed payload（参数结构体）定义执行域、超时、重试策略、输出抓取模式。
- 完善权限与安全防线：
  - 执行命令执行前做白名单校验与参数白名单过滤。
  - 增加错误码与标准化异常返回（非零返回、超时、权限拒绝三类可区分）。
- 运行时改造：
  - CLI 执行从事件总线触发，保持 Agent 逻辑与执行器解耦。
  - 命令执行日志写入可追踪字段（任务 ID、runtime mode、invocation trace）。

### 3）增强：知识库检索能力
- 重构知识检索入口为显式检索管道：
  - 关键词预检 → 向量召回 → 结果重排序 → 上下文打包。
- 引入检索评分可观测化：
  - 记录召回 score、rerank score、命中文档命中原因。
  - 支持按时间范围、标签、工作区、内容类型做二次过滤。
- 增强索引回填效率：
  - 生成时先做 embedding 缓存，查询时复用热缓存。
  - 支持分页增量更新与分片重建，避免全库阻塞。
- 支持 Agent 在创作页自动拉取知识引用块，降低 hallucination，并把引用源回填到任务元数据。

### 发版建议（执行层）
- AI 侧：将模型与生成链路统一放到 runtime skill/prompt 边界，避免在页面里写关键词路由。
- 生成侧：优先复用统一 `runtime:*` IPC + `bridge/ipcRenderer.ts`，避免分散调用。
- 性能建议：
  - 生图/生视频任务采用异步队列 + 并发阈值（默认 2/CPU）；
  - 知识检索开启短路缓存（最近 30s 热查询）与 TTL 保护；
  - 大列表分页加载，优先展示 skeleton + 增量刷新，保持首屏不被 IPC 阻塞。
