# Renderer Internationalization

RedBox desktop uses a lightweight renderer-side i18n layer in `src/i18n.tsx`.

## Architecture

- `I18nProvider` owns the active language and persists it to `localStorage`.
- `useI18n()` exposes `language`, `setLanguage`, and `t(key, params)`.
- Dictionaries are flat key/value maps. `zh-CN` is the source dictionary, and `en-US` must contain the same keys.
- The document `<html lang>` attribute updates when the language changes.

## Implementation Rules

- Use the built-in i18n layer for app shell, settings, and shared UI text.
- Keep runtime data, model names, file names, release notes, and user-authored content untranslated.
- Add keys to `zhCN` first, then mirror them in `enUS`.
- Use interpolation for small dynamic labels, for example `t('layout.currentVersion', { version })`.
- Do not add a large third-party i18n library unless pluralization, locale-aware formatting, or remote translation loading becomes necessary.

## Current Scope

The first supported languages are Simplified Chinese and English. The initial coverage includes the global app shell, sidebar, title bar, update dialog, YouTube capture prompt, settings navigation, the language selector, and high-frequency general settings labels.
