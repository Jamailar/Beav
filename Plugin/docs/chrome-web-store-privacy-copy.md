# Chrome Web Store Privacy Copy

用于 Chrome Web Store Developer Dashboard 的“隐私权规范”标签页。当前依据：

- `Plugin/src/manifest.json`
- `Plugin/src/background.js`
- `Plugin/src/browserControlBackground.js`
- `Plugin/src/background/*`

## 单一用途说明

Beav 浏览器扩展的单一用途是连接 Beav 桌面端，让用户把自己选择的网页、链接、选中文字和媒体内容保存到本机 Beav，并在用户从 Beav 明确发起浏览器任务时读取或操作对应网页。扩展不修改浏览器默认搜索、新标签页、主页或启动页，不注入广告，也不出售或共享用户数据。

## 远程代码说明

Beav 不加载 CDN 或第三方服务器托管的扩展脚本；扩展运行所需的 JavaScript、CSS 和资源都随扩展包提交。扩展会与本机 Beav 桌面端、Chrome Native Messaging host 和 localhost API 通信，用于接收用户在 Beav 中发起的网页采集或浏览器控制指令。浏览器控制任务可能通过 Chrome Debugger API 或 `chrome.scripting` 在用户指定或扩展创建的标签页中执行一次性的页面脚本，用于 DOM 读取、点击、输入、滚动、截图、剪贴板或文件选择等自动化操作。这些脚本只在用户明确发起任务时运行，不作为扩展持久代码加载，不来自公共 CDN，也不会暴露 Chrome 扩展 API 给网页。

## 主机权限说明

Beav 需要主机权限来读取用户当前选择保存或处理的网页内容、页面标题、链接、图片、视频和页面结构，并把结构化结果发送到本机 Beav 桌面端。`http://127.0.0.1` 和 `http://localhost` 权限用于连接本机 Beav 服务；`https://redbox.ziz.hk/*` 用于检查插件更新；小红书、Rednote、抖音、快手、TikTok、Bilibili、Reddit、X、Instagram、微信公众号、知乎等域名用于对应平台的内容采集适配。`<all_urls>` 用于“保存任意网页”和用户发起的通用浏览器控制任务，因为用户可能在任意网站上保存页面链接、摘录正文或让 Beav 处理当前网页。

## 权限说明

### activeTab

用于在用户点击扩展按钮、打开侧边栏或使用右键菜单时访问当前活动标签页，读取当前页面的标题、URL、选中内容和可保存状态。该权限只用于用户当前主动操作的标签页。

### alarms

用于后台定时任务，包括检查插件更新、本机 Native Messaging 连接重试、任务队列恢复和浏览器控制会话心跳清理，避免长时间任务中断后残留状态。

### bookmarks

用于在用户从 Beav 发起浏览器上下文查询时读取书签标题和 URL，帮助用户定位要保存或继续处理的网页。扩展仅读取书签元数据，不修改或删除书签。

### clipboardRead

用于用户明确请求时读取当前页面或浏览器剪贴板内容，例如把剪贴板文本、链接或图片保存到 Beav，或在浏览器控制任务中读取复制出来的页面内容。

### clipboardWrite

用于用户明确请求时把 Beav 生成、整理或采集得到的文本写入剪贴板，方便用户粘贴到网页表单、编辑器或其他应用。

### contextMenus

用于提供右键菜单入口，让用户从当前网页直接保存选中文字、链接、图片或当前页面到 Beav。

### debugger

用于用户从 Beav 明确发起的浏览器控制任务。通过 Chrome Debugger API，扩展可以对受控标签页执行 DOM 快照、点击、输入、滚动、截图、控制台日志读取、下载状态追踪、剪贴板授权和文件选择等操作。该能力仅用于用户指定的任务和标签页，会话结束后会释放控制状态。

### downloads

用于把用户选择保存的页面素材、图片、视频、导出文件或网页采集结果交给 Chrome 下载系统，并跟踪下载是否完成、失败或被取消。

### downloads.ui

用于在用户请求时打开或定位 Chrome 下载界面，方便用户查看 Beav 创建的下载任务结果。

### favicon

用于读取标签页 favicon，并在浏览器控制会话中给 Beav 管理的标签页显示轻量状态标记，帮助用户区分正在控制、待交付或已交付的标签页。

### history

用于在用户从 Beav 发起浏览器上下文查询时搜索最近访问的网页标题和 URL，帮助用户找回并保存最近浏览过的内容。扩展只读取有限数量的历史元数据，不修改历史记录。

### nativeMessaging

用于连接安装在用户本机的 Beav Native Messaging host，让扩展与 Beav 桌面端安全通信。该通道只在本机 Chrome 与本机 Beav 应用之间传输用户发起的采集、下载和浏览器控制请求。

### notifications

用于在采集、下载、更新检查或本机连接状态发生重要变化时通知用户，例如任务完成、任务失败或需要重新打开 Beav 桌面端。

### readingList

用于在用户从 Beav 发起浏览器上下文查询时读取 Chrome 阅读清单中的标题和 URL，帮助用户把稍后阅读的页面保存或继续处理。扩展仅读取阅读清单元数据。

### scripting

用于在用户选择保存或控制的页面中注入随扩展包提交的内容脚本，读取页面 DOM、提取结构化内容、显示 Beav 操作入口，或执行用户发起的点击、输入、滚动等页面自动化动作。

### sessions

用于在用户从 Beav 发起浏览器上下文查询时读取最近关闭的标签页和其他设备会话的有限元数据，帮助用户恢复并保存近期浏览内容。扩展不修改会话记录。

### sidePanel

用于提供 Beav 浏览器侧边栏，让用户查看当前页面识别结果、采集任务队列、采集历史、设置和浏览器控制连接状态。

### storage

用于保存扩展本地设置、任务队列状态、采集历史、Native Messaging 连接状态、浏览器控制会话状态和更新检查状态。数据保存在用户本机 Chrome 扩展存储中。

### tabGroups

用于把 Beav 创建或接管的浏览器控制标签页组织成临时标签组，便于用户识别任务相关页面；任务结束或释放标签页后会清理或恢复对应状态。

### tabs

用于读取标签页标题、URL、窗口、活动状态和标签组状态；创建、激活、导航、关闭用户明确发起的 Beav 受控标签页；以及把当前标签页内容保存到 Beav。

### topSites

用于在用户从 Beav 发起浏览器上下文查询时读取常访问网站的标题和 URL，帮助用户快速定位经常处理或保存的网页。扩展仅读取有限元数据。

### webNavigation

用于识别页面加载、跳转和 frame 结构，确保内容脚本注入到正确 frame，并在采集或浏览器控制任务中等待页面稳定后再读取 DOM 或执行动作。

## 数据使用确认文案

Beav 仅在用户明确操作或从 Beav 桌面端发起任务时处理网页数据。处理的数据可能包括当前页面 URL、标题、选中文字、页面正文、图片、视频链接、下载状态、浏览器标签页元数据、有限的历史/书签/阅读清单/常访问网站/最近关闭会话元数据，以及用户主动保存到 Beav 的内容。这些数据用于完成网页采集、下载、知识库保存和用户发起的浏览器控制任务。Beav 不出售用户数据，不将用户数据用于广告，不把用户数据转让给第三方，也不用于与扩展单一用途无关的目的。

## 上架风险备注

当前 manifest 同时申请 `<all_urls>`、`debugger`、`nativeMessaging`、`history`、`bookmarks`、`topSites`、`sessions` 等高敏权限。短期补齐上述说明可以解决表单缺项，但审核仍可能要求更窄权限。若公开商店版本被拒，建议做一个降权版：

1. 保留网页采集、侧边栏、下载、storage、contextMenus、scripting 和必要站点 host 权限。
2. 把 `debugger`、`nativeMessaging`、`history`、`bookmarks`、`topSites`、`sessions`、`tabGroups`、`webNavigation` 等浏览器控制能力拆成单独的高级版或企业/离线安装版。
3. 将 `<all_urls>` 改为更窄的 `optional_host_permissions`，在用户对某个网站启用采集或控制时再请求。
